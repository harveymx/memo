# 内部发布release beta 0.9,对前后端已经构建，但前端还是模拟数据为主的展示模式
* 打标签锁定beta(展示)版本 ； 给当前代码快照一个永久 ID，随时可回滚
```
git tag -a v0.9.0 -m "Beta release"
git push origin v0.9.0
```
* 切分支，以备热修复 ； 如果 beta 要独立修 bug，不影响 main 开发
```
git checkout -b release/v0.9
git push origin release/v0.9
```
* 回到main，继续开发 ； 保持主线向前，作为最新版本的基准
```
git checkout main
# 现在可以放心在 main 上开发 v1.0 了
```
